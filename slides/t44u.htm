UAL Bootstrap

A new approach to templating using Site Manager and Twitter Bootstrap

A tiny bit of context

  • UAL has been ‘on T4’ since summer 2011
  • Transitional period - complete redevelopment on the horizon
  • Moving from 6(7?) sites to 1 site
  • i.e. a core website with a consistent global design theme, which can be customised locally (with certain limitations).

New Approach

  • Requirement for a robust core framework for front-end design that can be modularised and extended throughout the system.
  • New technologies, trends, challenges: mobile, HTML5
  • Recent template development has progressed from old style to new style

Problem

  • Existing HTML + CSS is mix of legacy code, verbose CSS, no HTML5, weird grid system.
  • When building new templates in Site Manager, it’s not always possible to predict all potential use cases for a template.
  • Once templates are adopted, it is difficult to switch and ‘rationalise’.

Solution

  • A more modular approach to templating.
  • Create templates for irreducible components of an HTML page, and build up from there as patterns emerge.

The UAL Bootstrap

  • Adapted from the Twitter Bootstrap framework
  • Development environment is version controlled using git
  • Hosted (privately) on Github Pages
  • Uses Compass to compile minified CSS from SASS
  • Uses Jekyll to generate static HTML from a modular system of layouts and includes
  • Semantic HTML5 (naturally)

In Site Manager

  • Use a single style for everything!
  • One bootstrap.css file to update in the Media Library
  • CSS and Javascript are referenced using Related Content Branch
  • Commonly-used structural components (containers, rows etc) are mirrored into new sections to provide a scaffold for content

Advantages

  • Rapid prototyping both locally and in Site Manager
  • No need to commit to a template class while designs are still fluid
  • New CSS can reskin existing HTML without modifying the page structure
  • Easy to add new classes to elements on a per-section basis

Disadvantages

  • Not ideal for beginners
  • Pages can have 50+ content items - which becomes difficult to manage (correct div nesting can be v tricky)

Disadvantages

  • Templates (with minor variations) tend to proliferate without governance e.g.
    1. Code Only Template (71 instances)
    2. Code/Plain Text (490 instances)
    3. UAL: Plain Text/Code (2 instances)
